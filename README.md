# Задание 1  

Ответьте на вопрос в свободной форме.   
Чем частное облако отличается от общедоступного, публичного и гибридного?  

### Выполнение

### Публичное облако      
Инфраструктура принадлежит провайдеру и предоставляется множеству клиентов.  
Ресурсы распределяются между всеми пользователями.    
Примеры: AWS, Google Cloud, Yandex Cloud. Преимущества: низкая стоимость, быстрое масштабирование.   
Недостатки: меньше контроля над безопасностью, данные хранятся вместе с данными других клиентов.  
### Частное облако  
Инфраструктура используется только одной организацией. Может размещаться в собственном дата-центре компании или у провайдера, но полностью изолировано.   
Полный контроль над данными и настройками безопасности.  
Дороже публичного облака, но обеспечивает максимальную конфиденциальность.   
Используется банками, государственными организациями, медицинскими учреждениями.  
### Гибридное облако  
Комбинация частного и публичного облака. Критичные данные и приложения размещаются в частном облаке, менее критичные или требующие большой масштабируемости переносятся в публичное.   
Обеспечивает баланс между безопасностью, гибкостью и стоимостью.   
Требует настройки взаимодействия между облаками.  


# Задание 2  

Что обозначают: IaaS, PaaS, SaaS, CaaS, DRaaS, BaaS, DBaaS, MaaS, DaaS, NaaS, STaaS?   
Напишите примеры их использования.  

### Выполнение   

### IaaS (Infrastructure as a Service)    
Предоставление виртуализированных вычислительных ресурсов: серверы, хранилища, сети.    
Пользователь управляет ОС, приложениями, данными. Провайдер отвечает за физическое оборудование и виртуализацию.  
Примеры: аренда виртуальных машин (Yandex Compute Cloud, AWS EC2), виртуальные сети, блочное хранилище.  
### PaaS (Platform as a Service)  
Платформа для разработки и запуска приложений. Провайдер управляет инфраструктурой и ОС, разработчик только пишет код и загружает приложение.  
Примеры: Heroku, Google App Engine, Microsoft Azure App Service. Используется для быстрой разработки веб-приложений без настройки серверов.  
### SaaS (Software as a Service)  
Готовое программное обеспечение, доступное через интернет. Пользователь только использует приложение, всем остальным управляет провайдер.  
Примеры: Gmail, Microsoft 365, Salesforce, Яндекс.Диск, 1С в облаке.   
### CaaS (Containers as a Service)  
Платформа для запуска контейнеризированных приложений. Провайдер управляет оркестрацией контейнеров.  
Примеры: Kubernetes as a Service (Yandex Managed Service for Kubernetes, Google Kubernetes Engine), Docker-платформы.  
### DRaaS (Disaster Recovery as a Service)  
Облачное решение для восстановления данных и систем после сбоев или катастроф. Автоматическое резервное копирование и быстрое восстановление.  
Примеры: сервисы аварийного восстановления AWS, Azure Site Recovery. Используется для обеспечения непрерывности бизнеса.  
### BaaS (Backend as a Service)  
Готовый backend для мобильных и веб-приложений. Включает базы данных, аутентификацию, push-уведомления, облачное хранилище.  
Примеры: Firebase, AWS Amplify, Parse. Используется для быстрой разработки мобильных приложений без создания серверной части.  
### DBaaS (Database as a Service)  
Управляемые базы данных в облаке. Провайдер отвечает за установку, настройку, резервное копирование, обновления.  
Примеры: Yandex Managed Service for PostgreSQL, Amazon RDS, Google Cloud SQL.  
### MaaS (Monitoring as a Service)  
Облачный мониторинг IT-инфраструктуры. Сбор метрик, логов, оповещения о проблемах.  
Примеры: Datadog, New Relic, Yandex Monitoring. Используется для отслеживания производительности приложений и серверов.  
### DaaS (Desktop as a Service)  
Виртуальные рабочие столы в облаке. Пользователи получают доступ к рабочему месту через интернет с любого устройства.  
Примеры: Amazon WorkSpaces, Microsoft Windows Virtual Desktop, Citrix Virtual Apps and Desktops.  
### NaaS (Network as a Service)  
Сетевые сервисы в облаке: виртуальные сети, балансировщики нагрузки, VPN, фаерволы.  
Примеры: виртуальные сети в облачных провайдерах, SD-WAN решения.  
### STaaS (Storage as a Service)  
Облачное хранилище данных с оплатой за объем использования.  
Примеры: Yandex Object Storage, Amazon S3, Google Cloud Storage. Используется для хранения файлов, бэкапов, архивов.  

# Задание 3

Ответьте на вопрос в свободной форме.   
- Напишите, какой вид сервиса предоставляется пользователю в ситуациях:  
1) Всеми процессами управляет провайдер.  
2) Вы управляете приложением и данными, остальным управляет провайдер.  
3) Вы управляете операционной системой, средой исполнения, данными, приложениями, остальными управляет провайдер.  
4) Вы управляете сетью, хранилищами, серверами, виртуализацией, операционной системой, средой исполнения, данными, приложениями.

### Выполнение   

### Ситуация 1: Всеми процессами управляет провайдер.  
Это модель SaaS.    
Пользователь только использует приложение, не управляет инфраструктурой, платформой или приложением.  
### Ситуация 2: Вы управляете приложением и данными, остальным управляет провайдер.    
Это модель PaaS.  
Провайдер предоставляет платформу (ОС, среду выполнения, middleware), разработчик разворачивает свое приложение и данные.  
### Ситуация 3: Вы управляете операционной системой, средой исполнения, данными, приложениями, остальным управляет провайдер.  
Это модель IaaS.  
Провайдер предоставляет виртуальные машины, сети, хранилища. Пользователь устанавливает и настраивает все остальное.  
### Ситуация 4: Вы управляете сетью, хранилищами, серверами, виртуализацией, операционной системой, средой исполнения, данными, приложениями.  
Это традиционная on-premise инфраструктура (не облачный сервис).  
Компания владеет и управляет всем стеком от физического оборудования до приложений.
